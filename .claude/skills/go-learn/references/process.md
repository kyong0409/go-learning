# Go 학습 프로세스 상세 가이드

이 문서는 go-learn 스킬의 학습 프로세스를 상세히 정의합니다.

---

## 전체 학습 사이클

```
Phase 시작
  |
  v
[레슨 1] -> [레슨 2] -> ... -> [레슨 N]
  |            |                   |
  각 레슨:     각 레슨:             각 레슨:
  Theory       Theory              Theory
  Practice     Practice            Practice
  Quiz         Quiz                Quiz
  Feedback     Feedback            Feedback
  Feynman      Feynman             Feynman
  |
  v
[과제 1] -> [과제 2] -> ... -> [과제 M]
  |            |                   |
  각 과제:     각 과제:             각 과제:
  Brief        Brief               Brief
  코드 작성     코드 작성            코드 작성
  채점          채점                채점
  피드백        피드백               피드백
  리뷰          리뷰                리뷰
  |
  v
[프로젝트] (선택)
  |
  v

```

## 레슨 단계별 상세

### Theory (이론) — STOP 턴

**목표:** THEORY.md의 핵심 개념을 이해한다.

**진행:**
1. THEORY.md를 읽고 핵심을 추출한다
2. 사용자에게 구조화된 설명을 제공한다:
   - 이 레슨에서 배울 것 (bullet 3-5개)
   - 핵심 개념의 간략한 소개
   - 이전 레슨과의 연결점
   - Python/Java 경험자를 위한 비교 (해당되는 경우)
3. THEORY.md 직접 읽기를 안내한다
4. **STOP** (턴 종료)

**출력 형식:**
```markdown
## Lesson {N}: {title}

### 이 레슨에서 배울 것
- {concept 1}
- {concept 2}
- {concept 3}

### 핵심 개념 미리보기
{간략한 개념 소개}

### 이전 레슨과의 연결
{lesson N-1}에서 배운 {concept}가 여기서 {how}으로 확장됩니다.

---
THEORY.md를 직접 읽어보세요.
파일 위치: `go-curriculum/{path}/THEORY.md`
다 읽으시면 '다음'이라고 해주세요.
```

### Practice (실습) — STOP 턴

**목표:** main.go를 실행하고 코드를 이해한다.

**진행:**
1. main.go (또는 예제 파일들)를 읽는다
2. 코드의 핵심 포인트를 설명한다:
   - 주요 함수/타입의 역할
   - 이론에서 배운 개념의 실제 구현
   - 주목할 Go 관용 표현
3. 실행 방법을 안내한다
4. 코드 실험 제안을 1-2개 던진다
5. **STOP** (턴 종료)

**실험 제안 예시:**
- "main.go에서 {variable}의 값을 바꿔보세요. 어떤 결과가 나오나요?"
- "{function}을 호출할 때 {다른 인자}를 넣으면 어떻게 될까요?"
- "에러 처리를 제거하면 어떤 일이 벌어지나요?"

---

### Quiz (퀴즈 채점)

**목표:** 이론과 실습 내용의 이해도를 평가한다.

**퀴즈 생성 규칙:**
1. THEORY.md의 핵심 개념에서 출제한다
2. 3문제: 기본(1) + 응용(1) + 심화(1)
3. 각 문제는 4개 선택지 (정답 1 + 그럴듯한 오답 3)
4. 코드 스니펫이 포함된 문제를 최소 1개 포함
5. 단순 암기가 아닌 이해를 확인하는 문제

**문제 유형:**
- **기본:** "다음 중 {concept}에 대한 설명으로 올바른 것은?"
- **응용:** "다음 코드의 출력 결과는?" (코드 스니펫 제공)
- **심화:** "다음 상황에서 가장 적절한 Go 관용 패턴은?"

**진행:** AskUserQuestion으로 한 문제씩 출제하고, 각 문제 후 즉시 정답/오답 알림.

---

### Feedback (피드백)

**목표:** 퀴즈 결과를 분석하고 부족한 부분을 보충한다.

**진행:**
1. 전체 점수를 알려준다 (예: "3/3 정답! 완벽합니다" 또는 "2/3 정답")
2. 오답 문제에 대해:
   - 왜 선택한 답이 틀렸는지 설명
   - 정답이 왜 맞는지 설명
   - 관련 THEORY.md 섹션 참조
3. 이 레슨의 핵심 takeaway를 3줄로 정리:
   ```
   핵심 정리:
   1. {takeaway 1}
   2. {takeaway 2}
   3. {takeaway 3}
   ```

### Feynman Review (파인만 검토)

**목표:** 파인만 테크닉으로 깊은 이해를 검증한다.

**핵심 원칙:** 사용자가 스스로 답을 떠올리도록 유도한다. **절대 답을 바로 알려주지 않는다.**

**진행:**
1. 사용자에게 설명을 요청한다:
   > "이 레슨의 핵심을 자신의 말로 설명해보세요.
   > 프로그래밍을 모르는 친구에게 설명한다고 생각하고,
   > 전문 용어 없이 말해보세요."

2. 사용자의 설명을 듣고 평가한다:
   - **정확한 설명**: "맞습니다! {구체적으로 잘 설명한 부분}을 잘 짚었네요."
   - **추가 질문이 필요한 부분**: "좋은 설명이에요. 그런데 {concept}가 왜 그렇게 동작하는지 조금 더 설명해줄 수 있나요?"
   - **오류**: "거의 맞는데, {part}는 조금 다릅니다. 다시 한번 생각해볼까요?"
   - **빠진 핵심**: 아래 힌트 유도 프로토콜을 따른다.

3. **힌트 유도 프로토콜** (사용자가 "모르겠다/기억이 안나" 또는 핵심을 빠뜨렸을 때):

   절대 바로 답을 알려주지 않는다. 단계적으로 유도한다:

   **1차 유도 — 넓은 힌트 (방향만 제시):**
   - 퀴즈에서 다뤘던 문제를 환기시킨다
   - 레슨의 어떤 영역이 빠졌는지만 알려준다
   - 예: "퀴즈에서 import 관련 문제가 있었는데 기억나세요?"
   - 예: "이 레슨에서 다른 언어와 비교한 부분이 있었는데..."

   **2차 유도 — 좁은 힌트 (구체적 단서):**
   - 빠진 개념의 맥락을 좁혀준다
   - 예: "안 쓰는 패키지를 import하면 다른 언어와 달리 어떻게 된다고 했죠?"
   - 예: "중괄호 { 위치에 대한 규칙이 있었는데, 왜 필요했을까요?"

   **3차 유도 — 핵심 키워드 포함 단서:**
   - 거의 직접적인 단서를 준다
   - 예: "Go는 '컴파일 에러'로 어떤 것들을 강제했었죠?"
   - 예: "세미콜론이 자동으로 삽입되면서 생기는 제약이 있었는데..."

   **4차 (최종) — 보충 설명 후 통과:**
   - 3회 유도에도 떠올리지 못하면, 답을 알려주되 왜 중요한지 강조한다
   - 통과 처리하되, 해당 개념이 다음 레슨에서 나오면 다시 짚어줄 것을 안내한다

4. 통과 판정:
   - 핵심 개념의 70% 이상을 자신의 말로 정확히 설명 -> 통과
   - 미통과 시: 힌트 유도 프로토콜을 따라 최대 3회 재시도
   - 4회차에도 미흡하면: 보충 설명한 후 통과 처리

5. 통과 시 격려와 함께 다음 단계 안내

## 과제 단계별 상세

### Assignment Brief — STOP 턴

1. README.md를 읽고 과제 개요를 설명
2. 난이도, 예상 소요 시간, 연결 레슨을 안내
3. 구현 파일의 TODO 부분을 보여줌
4. **STOP** (사용자가 코드 작성)

### Grading (채점)

1. `go test ./...`를 실행
2. `-v` 플래그로 상세 결과 확인
3. 통과/실패 테스트를 구분하여 보여줌
4. 점수 = (통과 테스트 수 / 전체 테스트 수) * 100

### Feedback (피드백)

1. 실패 테스트별 힌트 제공 (정답 코드 절대 미제공)
2. 힌트 수준 조절:
   - 1차: 어떤 개념이 필요한지만 안내
   - 2차 (재요청 시): 더 구체적인 방향 제시
   - 3차 (재요청 시): 유사한 코드 패턴 예시 제공
3. 모든 테스트 통과 시 -> Review 단계로

### Review (리뷰)

1. solution/ 참고 풀이와 비교
2. 차이점 분석 (접근 방식, 코드 스타일, 성능)
3. Go 관용 표현 관점의 개선점 제안
4. 잘한 점 칭찬

## 네비게이션 명령

사용자가 언제든 사용할 수 있는 명령:

| 명령 | 동작 |
|------|------|
| "현황" / "status" | 전체 진행 상황 표시 |
| "다음" / "next" | 다음 단계로 진행 |
| "건너뛰기" / "skip" | 현재 단계 건너뛰기 (Feynman 포함) |
| "과제" / "assignment" | 과제 목록 및 시작 |
| "프로젝트" / "project" | 현재 Phase 프로젝트 |
| "Phase N" / "phase N" | 특정 Phase로 이동 |
| "Lesson N" / "lesson N" | 현재 Phase의 특정 레슨으로 이동 |
| "도움" / "help" | 사용 가능한 명령 목록 |
