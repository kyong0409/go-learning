# 채점 가이드

이 문서는 go-learn 스킬의 채점 기준과 방법을 정의합니다.

---

## 레슨 퀴즈 채점

### 점수 체계
- 3문제 출제 (기본 1 + 응용 1 + 심화 1)
- 각 문제 정답: 1점 / 오답: 0점
- 만점: 3점 (100%)
- 기록: `scores.phase{N}.quizzes.lesson{M}` = 정답 수 / 3

### 등급
| 점수 | 등급 | 의미 |
|------|------|------|
| 3/3 | A | 완벽한 이해 |
| 2/3 | B | 양호, 오답 개념 복습 권장 |
| 1/3 | C | 부족, 이론 재학습 권장 |
| 0/3 | D | 이론 다시 읽기 필요 |

### 퀴즈 생성 가이드

**좋은 퀴즈 문제:**
- 단순 암기가 아닌 이해를 확인
- 코드 실행 결과를 예측하는 문제 포함
- 오답 선택지가 흔한 오해를 반영

**나쁜 퀴즈 문제:**
- "다음 중 Go 키워드가 아닌 것은?" (단순 암기)
- 너무 모호하거나 트릭성 문제
- 레슨 범위를 벗어나는 문제

## 과제 채점

### 점수 체계
- `go test ./...` 실행 결과 기반
- 점수 = (통과 테스트 수 / 전체 테스트 수) * 100
- 기록: `scores.phase{N}.assignments.{name}` = 점수

### 채점 실행
```bash
cd go-curriculum/{phase_dir}/assignments/{assignment_dir}
go test ./... -v
```

### 피드백 수준 단계

사용자가 테스트를 통과하지 못했을 때:

**Level 1 (기본 힌트):**
> "이 테스트는 {무엇을} 확인합니다. {어떤 개념}을 다시 확인해보세요."

**Level 2 (방향 제시):**
> "이 부분은 {구체적 접근법}으로 해결할 수 있습니다. Lesson {N}의 {section}을 참고해보세요."

**Level 3 (패턴 예시):**
> "비슷한 패턴의 예시입니다: {유사 코드 패턴}. 이 패턴을 과제에 적용해보세요."

**정답 코드는 절대 제공하지 않는다.** solution/은 모든 테스트 통과 후에만 참조한다.

## 파인만 검토 평가

### 평가 기준
파인만 검토는 점수가 아닌 **통과/미통과**로 평가한다.

**통과 조건 (모두 충족):**
- [ ] 핵심 개념의 70% 이상을 자신의 말로 설명
- [ ] 전문 용어 없이 (또는 풀어서) 설명 가능
- [ ] 개념 간 관계를 이해하고 있음

**미통과 시:**
- 부족한 부분을 구체적으로 짚어준다
- 다시 설명을 요청한다 (최대 2회 재시도)
- 3회차에도 미흡하면 보충 설명 후 통과 처리

### 파인만 검토 질문 예시

**좋은 요청:**
> "Go의 슬라이스가 무엇인지, 배열과 어떻게 다른지 설명해보세요.
> 비유를 써서 설명하면 더 좋습니다."

**추가 심화 질문:**
> "좋은 설명이에요. 그렇다면 슬라이스의 '용량(capacity)'은 왜 필요한 걸까요?"

## 전체 진행 통계

### Phase 완료 조건
- 모든 레슨 완료 (퀴즈 + 파인만 검토 통과)
- 모든 과제 완료 (전 테스트 통과)
- 프로젝트는 선택 사항

### 전체 등급 (28개 과제 기준)
| 총점 범위 | 등급 | 의미 |
|-----------|------|------|
| 90%+ | S | K8s 컨트리뷰터 준비 완료 |
| 80-89% | A | 프로덕션 레벨 |
| 70-79% | B | 실무 투입 가능 |
| 60-69% | C | 기초는 탄탄, 심화 필요 |
| 60% 미만 | D | 기초 복습 권장 |

### 기존 grade.sh 활용
go-curriculum에는 자동 채점 스크립트가 있다:
```bash
bash go-curriculum/grade.sh          # 전체 채점
bash go-curriculum/grade.sh phase1   # Phase별 채점
```
이 스크립트를 활용하여 전체 현황을 빠르게 파악할 수 있다.
