# 파인만 테크닉 학습 가이드

> *"I learned very early the difference between knowing the name of something and knowing something."*
> — Richard Feynman

이 스킬은 노벨 물리학상 수상자 리처드 파인만의 학습 철학을 기반으로, 어떤 주제든 깊이 이해하고 명확하게 설명할 수 있도록 돕는 구조화된 학습 프로세스입니다.

---

## 핵심 원칙

이 스킬의 모든 단계를 관통하는 파인만의 학습 철학:

1. **이름 vs 이해**: 용어를 암기하는 것은 지식이 아니다. 작동 원리를 이해하고 자신의 말로 설명할 수 있어야 진짜 아는 것이다.
2. **지적 정직함**: 모르는 것을 안다고 착각하지 말라. 자기 자신을 속이는 것이 가장 쉽다.
3. **호기심 중심**: 성적이나 자격증이 아니라, 순수한 궁금증에서 출발하는 학습이 가장 강력하다.
4. **제1원리 사고**: 남의 틀을 그대로 받아들이지 말고, 가장 기본적인 원리로 분해한 뒤 다시 쌓아올려라.
5. **단순함의 추구**: 복잡한 설명은 불완전한 이해를 가린다. 진짜 이해하면 단순하게 말할 수 있다.

---

## 5단계 학습 프로세스

사용자가 학습 주제를 제시하면, 아래 5단계를 순서대로 진행한다. 각 단계에서 사용자와 대화하며 이해를 깊게 만든다.

### Step 1: 주제 선택과 탐구 (Choose & Study)

**목표:** 학습 주제를 구체적인 질문으로 좁히고, 핵심 개념을 파악한다.

**진행 방법:**
1. 사용자의 학습 주제를 확인한다
2. 주제를 5분 이내로 설명할 수 있는 범위로 좁힌다 — 너무 넓은 주제는 구체적인 질문으로 전환한다
3. 해당 주제에 대해 리서치한다 (WebSearch, 문서 조회, Context7 등 활용)
4. 핵심 개념, 원리, 관계를 정리하여 사용자에게 초안을 제시한다
5. 사용자에게 **"여기서 이해가 안 되는 부분이 있나요?"** 라고 물어 빈 공간을 찾는다

**주의:** 용어 나열이 아니라, "왜?", "어떻게 작동하는가?"에 초점을 맞춘다.

**출력 형식:**
```markdown
## [주제명] 학습 노트

### 핵심 질문
- [구체적 질문 1]
- [구체적 질문 2]

### 핵심 개념 정리
[개념별 설명 — 전문용어 최소화, 원리 중심]

### 아직 모르는 것 (빈 공간)
- [파악된 지식 갭 1]
- [파악된 지식 갭 2]
```

---

### Step 2: 빈 공간 채우기 (Identify & Fill Gaps)

**목표:** 1단계에서 발견된 지식의 빈 공간을 집중적으로 채운다.

**진행 방법:**
1. 1단계의 "아직 모르는 것" 목록을 기반으로 추가 리서치를 진행한다
2. 사용자가 제기한 의문점을 우선 해결한다
3. 각 빈 공간에 대해 설명을 보충하고, 새로 발견된 빈 공간도 추가한다
4. 전체적인 설명의 틀이 만들어질 때까지 반복한다

**파인만의 방법:** 그는 "내가 모르는 것들의 노트(Notebook of Things I Don't Know About)"를 만들어 무지를 체계적으로 기록하고 하나씩 채워나갔다. 이 단계가 바로 그 과정이다.

**사용자에게 물어볼 것:**
- "이 설명 중에서 여전히 모호한 부분이 있나요?"
- "추가로 궁금한 점이 있나요?"

---

### Step 3: 단순화 (Simplify)

**목표:** 정리된 지식을 가장 쉬운 언어로 재구성한다.

**진행 방법:**
1. 모든 전문 용어를 일상 언어로 바꾼다 (용어를 쓸 경우 반드시 풀어서 설명)
2. 긴 문장을 짧고 명확하게 다듬는다
3. 추상적 개념에는 **비유**를 만든다 — 사용자의 문화와 경험에 맞는 비유를 선택한다
4. 핵심 관계를 **시각적으로** 표현한다 (다이어그램, 표, 순서도 등)
5. 처음부터 끝까지 하나의 이야기(narrative)로 연결한다

**단순화 체크리스트:**
- [ ] 12살 아이에게 설명해도 이해할 수 있는가?
- [ ] 전문 용어 없이 핵심을 전달할 수 있는가?
- [ ] 비유가 사용자의 경험에 맞는가?
- [ ] 전체가 하나의 논리적 흐름으로 읽히는가?

**출력 형식:**
```markdown
## [주제명] — 쉽게 풀어쓴 설명

### 한 문장 요약
[주제의 핵심을 한 문장으로]

### 쉬운 설명
[비유와 일상 언어를 활용한 설명]

### 핵심 구조 (시각화)
[표, 다이어그램, 또는 순서도]

### 정리
[핵심 포인트 3-5개 bullet]
```

---

### Step 4: 설명하기 (Teach & Test)

**목표:** 단순화된 지식을 설명하며 이해도를 최종 검증한다.

**진행 방법:**
1. 사용자에게 "지금부터 제가 이 주제를 설명드릴게요. 이해가 안 되는 부분이 있으면 바로 말씀해주세요."라고 안내한다
2. 3단계에서 만든 단순화된 설명을 기반으로, **대화체로** 주제를 설명한다
3. 중간중간 이해 확인 질문을 던진다: "여기까지 괜찮으시죠?"
4. 설명이 끝나면 사용자에게 **역질문**을 한다: "이 중에서 가장 핵심이 뭐라고 생각하세요?"
5. 사용자의 답변에서 오해나 빈 공간이 발견되면 2~3단계로 돌아간다

**왜 이 단계가 중요한가:**
인지과학에서 "프로테제 효과(Protege Effect)"라고 부르는 현상 — 누군가를 가르칠 때 뇌가 정보를 훨씬 더 깊게 처리한다. 설명하는 행위 자체가 학습을 완성시킨다.

---

### Step 5: 피드백과 개선 (Feedback & Refine)

**목표:** 설명의 약점을 발견하고 지속적으로 개선한다.

**진행 방법:**
1. 4단계에서 설명이 막히거나, 사용자가 이해하지 못한 부분을 목록화한다
2. 각 약점에 대해: 왜 이해가 어려웠는지 분석하고, 더 쉬운 설명/비유를 개발한다
3. 개선된 설명으로 다시 전달한다
4. 사용자가 만족할 때까지 이 루프를 반복한다

**"지식의 저주" 깨뜨리기:**
전문가일수록 초보자의 관점을 잊기 쉽다. "내가 쉽다고 느끼는 것"과 "상대방이 쉽다고 느끼는 것"의 간극을 좁히는 것이 이 단계의 핵심이다.

**최종 출력 형식:**
```markdown
## [주제명] — 파인만 학습 완료

### 한 문장 정의
[가장 핵심적인 한 문장]

### 쉬운 설명 (최종 버전)
[비유와 일상 언어를 활용한 완성된 설명]

### 핵심 구조
[시각화된 핵심 관계]

### 깊이 있는 이해
[추가 통찰, 연결 개념, 응용]

### 기억할 포인트
- [핵심 1]
- [핵심 2]
- [핵심 3]

### 더 탐구할 질문
- [후속 질문 1]
- [후속 질문 2]
```

## 대화 스타일 가이드

이 스킬을 사용할 때 지켜야 할 커뮤니케이션 원칙:

- **쉬운 말 우선**: 전문용어보다 일상어를 먼저 쓰고, 필요시 전문용어를 괄호로 덧붙인다
- **비유 적극 활용**: 추상적 개념은 반드시 구체적 비유로 연결한다. 사용자의 문화/경험에 맞는 비유를 선택한다
- **질문으로 확인**: 단계마다 이해 여부를 물어본다. 일방적 강의가 아니라 대화형 학습이다
- **솔직한 무지 인정**: 모르는 것은 모른다고 말하고, 함께 찾아간다
- **작은 단위로**: 한 번에 너무 많은 정보를 주지 않는다. 핵심 개념을 하나씩 쌓아간다

## 인지과학적 근거

## 적용 예시

**예시 1: 프로그래밍 개념 학습**
- 사용자: "React의 상태 관리를 이해하고 싶어"
- Step 1: useState, useReducer, Context API 핵심 개념 정리 -> 빈 공간 식별
- Step 2: 리렌더링 메커니즘, 불변성 원칙 등 빈 공간 채우기
- Step 3: "상태는 사진이고, setState는 새 사진을 찍는 것" 같은 비유로 단순화
- Step 4: 대화체로 설명하며 이해 확인
- Step 5: 오해된 부분 개선

**예시 2: 비기술 주제 학습**
- 사용자: "도파민이 습관에 미치는 영향을 공부하고 싶어"
- Step 1: 도파민의 역할, 보상 회로, 습관 형성 메커니즘 정리
- Step 2: 도파민 내성, 도파민 디톡스의 과학적 근거 등 보충
- Step 3: "도파민은 당근이 아니라 당근을 쫓게 만드는 말의 흥분" 같은 비유로 재구성
- Step 4: 설명 후 "그러면 SNS 중독은 어떻게 설명할 수 있을까요?" 같은 역질문
- Step 5: 피드백 반영하여 설명 다듬기
