# 과제 A3: 구조화된 로그 분석기

**난이도**: ★★★★☆
**예상 소요 시간**: 4-5시간

## 과제 설명

`slog` 형식의 JSON 로그 파일을 읽어 분석하고 보고서를 생성하는 도구를 만드세요.

## 요구사항

### 기능 요구사항

1. **JSON 로그 파싱**: 한 줄씩 읽어 LogEntry 구조체로 파싱
2. **레벨별 집계**: DEBUG/INFO/WARN/ERROR 개수 집계
3. **시간별 집계**: 1시간 단위로 요청 수 집계
4. **오류 패턴 탐지**: 같은 오류 메시지가 5회 이상 반복되면 감지
5. **스트리밍**: 대용량 파일도 메모리 효율적으로 처리 (bufio.Scanner 사용)
6. **다중 파일**: 여러 파일을 동시에(concurrent) 처리
7. **보고서 생성**: 분석 결과를 텍스트 및 JSON 형식으로 출력

### JSON 로그 형식 (slog 표준)

```json
{"time":"2024-01-15T10:23:45Z","level":"INFO","msg":"서버 시작","port":8080}
{"time":"2024-01-15T10:23:46Z","level":"ERROR","msg":"DB 연결 실패","error":"timeout"}
{"time":"2024-01-15T10:23:47Z","level":"WARN","msg":"응답 지연","latency_ms":250}
```

### 출력 예시

```
=== 로그 분석 결과 ===
파일: app.log
총 엔트리: 150개

레벨별 분포:
  DEBUG:  12개 (8.0%)
  INFO:   98개 (65.3%)
  WARN:   25개 (16.7%)
  ERROR:  15개 (10.0%)

시간별 분포 (상위 5):
  10:00 ~ 11:00  45개
  11:00 ~ 12:00  67개
  12:00 ~ 13:00  38개

오류 패턴 (5회 이상):
  "DB 연결 실패"  8회
  "타임아웃"      6회
```

## 구현 파일

- `analyzer.go`: Analyzer 구조체, LogEntry 파싱, 집계 로직
- `report.go`: 보고서 생성 (텍스트/JSON)
