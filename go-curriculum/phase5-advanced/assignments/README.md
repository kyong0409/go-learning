# Phase 5 과제 모음

Phase 5에서 배운 고급 시스템 개념을 직접 구현해보는 과제들입니다.

## 과제 목록

| 과제 | 주제 | 핵심 기술 | 난이도 |
|------|------|-----------|--------|
| [A1 - 파일 검색 CLI](a1-cli-tool/) | Cobra CLI + 파일 시스템 | Cobra, 정규식, 재귀 탐색 | ★★★☆☆ |
| [A2 - HTTP 부하 테스트 도구](a2-load-tester/) | 동시성 + HTTP | goroutine, 채널, 통계 | ★★★★☆ |
| [A3 - 로그 분석기](a3-log-analyzer/) | 구조화된 로그 파싱 | JSON, 스트리밍, 패턴 분석 | ★★★★☆ |
| [A4 - 플러그인 시스템](a4-plugin-system/) | 인터페이스 + 위상 정렬 | Plugin 인터페이스, Kahn 알고리즘, 생명주기 | ★★★★☆ |
| [A5 - 분산 잠금 시스템](a5-distributed-lock/) | 분산 시스템 시뮬레이션 | 펜싱 토큰, TTL, 대기 큐, 데드락 감지 | ★★★★★ |

## 진행 방법

1. 각 과제 디렉터리의 `README.md`를 먼저 읽어 요구사항 파악
2. 골격 코드(`*.go`)를 보고 함수 시그니처 파악
3. 테스트를 먼저 실행해 현재 실패 확인: `go test ./... -v`
4. 구현 후 모든 테스트 통과 확인
5. `solution/` 디렉터리의 참고 풀이와 비교

## 채점 기준

각 과제의 `*_test.go` 파일을 실행하면 한국어 점수 보고서가 출력됩니다:

```bash
go test ./... -v -run TestGrade
```

출력 예시:
```
=== 채점 결과 ===
통과: 15/20
점수: 75/100
```

## 공통 규칙

- 외부 패키지 사용 가능 (단, 핵심 로직은 직접 구현)
- 테스트 파일 수정 금지
- `solution/` 디렉터리는 막힐 때만 참고
- 코드 주석은 한국어로 작성 권장
- A4, A5는 `-race` 플래그로 레이스 컨디션도 반드시 검사
