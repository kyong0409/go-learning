## 진행 방법

1. 각 과제 폴더의 `README.md`를 먼저 읽습니다.
2. 스켈레톤 파일(`.go`)의 함수 시그니처를 확인합니다.
3. 테스트 파일(`_test.go`)을 실행하여 현재 상태를 확인합니다.
4. 구현을 완성하고 모든 테스트를 통과시킵니다.
5. `solution/` 폴더의 참고 답안과 비교합니다.

## 채점 방법

```bash
# 각 과제 디렉토리에서 테스트 실행
go test ./... -v

# 채점 결과만 확인
go test ./... -v -run TestGrade

# 레이스 컨디션 감지 (A4, A5 권장)
go test -race ./...

# 점수는 통과한 테스트 수에 따라 자동 계산됩니다.
```

## 학습 목표

- **A1**: 제네릭 타입 파라미터와 제약조건 이해
- **A2**: HTTP 미들웨어 패턴과 동시성 구현
- **A3**: 완전한 REST API 설계 및 구현 경험
- **A4**: 메트릭 시스템 설계, atomic 연산, HTTP 텍스트 형식 출력
- **A5**: 제네릭 + `container/list` LRU, TTL 만료, 백그라운드 정리, 레이스 프리 설계
