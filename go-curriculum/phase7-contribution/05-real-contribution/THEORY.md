## 기여 전략: 단계적 접근

### Step 1: 커뮤니티 관계 구축 (1주)

1. Kubernetes Slack 가입 → `#agent-sandbox` 입장
2. 자기소개: "Go 학습 중이고, agent-sandbox에 기여하고 싶습니다"
3. SIG Apps 미팅에 참석 (격주 월요일)
4. 최근 PR들을 읽으며 코딩 스타일 파악

### Step 2: 코드베이스 탐색 (1주)

1. 리포지토리 포크 & 로컬 빌드
2. `make test` 전체 통과 확인
3. api/ → controllers/ 순서로 코드 읽기
4. E2E 테스트 실행하며 실제 동작 관찰

### Step 3: 첫 PR (1-2주)

1. Issue #331 (race detector) 도전
2. 또는 문서/typo 수정 (부담 없는 첫 PR)
3. 리뷰 피드백 반영하며 프로세스 체득

### Step 4: 본격 기여 (2-4주)

1. Issue #168 (E2E 테스트) 또는 #403 (벤치마킹)
2. 이슈에 코멘트로 의도 표명: "I'd like to work on this"
3. 설계 방향을 먼저 논의 (큰 변경의 경우)
4. 작은 PR로 나누어 제출

### Step 5: 지속적 기여

1. 새 이슈 발견하여 직접 제안
2. 다른 기여자의 PR 리뷰 참여
3. SIG 미팅에서 로드맵 논의 참여

## 실전 기여 시 주의사항

### Do's

- **작은 PR부터**: 한 PR에 한 가지 변경만
- **이슈 먼저 확인**: 누군가 이미 작업 중인지 확인
- **설계 먼저 논의**: 큰 변경은 이슈에서 방향을 합의
- **테스트 필수**: 코드 변경에는 반드시 테스트 동반
- **빠른 응답**: 리뷰 피드백은 24-48시간 이내 대응

### Don'ts

- **거대한 PR 금지**: 500줄+ PR은 리뷰받기 어려움
- **동시에 여러 이슈 금지**: 하나를 완료한 후 다음으로
- **포스 푸시 남용 금지**: 리뷰 히스토리가 사라짐
- **코딩 스타일 무시 금지**: 기존 코드와 일관되게
- **리뷰 없이 수정 금지**: 피드백 없는 커밋 추가는 혼란

## 기여 성과 측정

| 마일스톤 | 의미 |
|----------|------|
| 첫 PR 머지 | 기여 프로세스 체득 |
| 3개 PR 머지 | 코드베이스에 익숙해짐 |
| 이슈 제안 및 수정 | 프로젝트 방향에 기여 |
| 다른 PR 리뷰 | 커뮤니티 멤버로 인정 |
| SIG 미팅 발표 | 프로젝트 리더십 |

## 핵심 정리

1. 커뮤니티 참여 → 코드 탐색 → 첫 PR → 본격 기여 순서로 접근
2. Issue #331 (race detector)이 가장 현실적인 첫 기여
3. 작은 PR, 빠른 응답, 테스트 동반이 핵심 원칙
4. 코드 기여뿐 아니라 리뷰, 문서, 이슈 제안도 기여다
5. 꾸준함이 가장 중요하다 — 한 달에 1 PR이라도 지속
