# A4: 실전 기여 — PR 제출 (★★★★★)

## 목표

agent-sandbox에 실제 PR을 제출하고 머지를 목표로 한다.

## 캡스톤 과제

이것은 전체 커리큘럼의 최종 과제입니다. Phase 1-6에서 배운 모든 것과 Phase 7의 기여 프로세스를 종합하여, **실제 오픈소스 프로젝트에 코드를 기여**합니다.

## 요구사항

### 1. 이슈 선택
아래 중 하나를 선택하거나, 직접 이슈를 발견하여 제안:

**추천 이슈:**
| 이슈 | 난이도 | 영역 |
|------|--------|------|
| #331 (race detector) | ★★★☆☆ | 테스트 인프라 |
| #168 (E2E 테스트) | ★★★½☆ | E2E 테스트 |
| #403 (벤치마킹) | ★★★★☆ | 성능 테스트 |
| #166 (웹사이트) | ★★★☆☆ | 문서/프론트엔드 |
| 자체 발견 이슈 | 가변 | 가변 |

### 2. 사전 준비
- [ ] 이슈에 코멘트: "I'd like to work on this. Here's my approach: ..."
- [ ] 메인테이너의 피드백 확인
- [ ] (큰 변경의 경우) 설계 방향 합의

### 3. 구현
- [ ] 기능 브랜치 생성
- [ ] 코드 구현
- [ ] 테스트 작성/수정
- [ ] `make build && make test && make lint` 통과

### 4. PR 제출
- [ ] 커밋 메시지에 `Signed-off-by` 포함
- [ ] PR 설명에 이슈 참조 (`Fixes #NNN`)
- [ ] 변경 사항 설명 (What / Why / How)
- [ ] 테스트 방법 설명

### 5. 리뷰 대응
- [ ] CI 통과 확인
- [ ] 리뷰 피드백 반영
- [ ] 추가 수정 커밋
- [ ] 최종 승인 & 머지

## 평가 기준

| 항목 | 배점 |
|------|------|
| 이슈 선택 및 사전 소통 | 10 |
| 코드 품질 (Go 관용 표현, 일관성) | 25 |
| 테스트 동반 여부 | 20 |
| PR 설명 품질 | 10 |
| 리뷰 피드백 대응 | 15 |
| 실제 머지 성공 | 20 |
| **합계** | **100** |

## 참고

- PR이 머지되지 않더라도, 과정 자체가 학습입니다
- 리뷰 피드백은 최고의 Go 코드 리뷰 경험입니다
- 한 번에 완벽할 필요 없습니다 — 반복하며 개선하세요
- 기여 이력은 GitHub 프로필에 영구적으로 남습니다

## 축하합니다!

이 과제를 완료하면 당신은:
- Go 언어를 기초부터 프로덕션 수준까지 마스터했고
- Kubernetes 컨트롤러 패턴을 이해하고 구현할 수 있으며
- 실제 CNCF 프로젝트에 기여한 오픈소스 컨트리뷰터입니다

Phase 1의 `fmt.Println("Hello, World!")`에서 시작하여
Kubernetes 생태계에 코드를 기여하기까지 — 대단한 여정이었습니다.
